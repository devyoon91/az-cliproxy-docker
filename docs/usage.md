## 기본 조작

### UI 액션 버튼

채팅 입력창 아래에 위치한 액션 버튼들:

| 버튼 | 기능 |
|------|------|
| **Pause/Resume Agent** | 에이전트 일시정지/재개 |
| **Import Knowledge** | 지식베이스에 파일 추가 (.txt, .pdf, .csv, .html, .json, .md) |
| **File Browser** | 파일 업로드/다운로드/삭제/탐색 |
| **Context** | LLM에 전송되는 전체 컨텍스트 확인 (디버깅용) |
| **History** | 대화 히스토리 JSON 확인 |
| **Nudge** | 에이전트가 멈췄을 때 마지막 프로세스 재시작 |
| **Restart** | 프레임워크 전체 재시작 (채팅 히스토리 유지) |

### 파일 첨부

- 채팅 입력창 옆 📎 아이콘 클릭
- 여러 파일 동시 첨부 가능
- 이미지는 썸네일 미리보기 제공
- "이 파일들을 /work_dir/project에 옮겨줘" 같은 지시 가능

### 파일 브라우저

- 디렉토리 탐색 (Up 버튼으로 상위 이동)
- 파일/폴더 업로드, 다운로드, 삭제
- 폴더는 zip으로 다운로드
- work_dir이 기본 작업 디렉토리

## 도구 사용법

Agent Zero의 핵심 도구들:

### code_execution_tool (코드 실행)
Python, Node.js, Bash 코드를 직접 실행합니다.
```
"Python으로 데이터 분석하고 차트 만들어줘"
```

### search_engine (웹 검색)
내장 SearXNG을 통해 웹 검색을 수행합니다.
```
"최신 React 19 변경사항 검색해줘"
```

### document_query (문서 분석)
PDF, HTML, 웹페이지의 내용을 읽고 분석합니다.
```
"이 PDF에서 매출 데이터를 추출해줘"
```

### browser_agent (브라우저 자동화)
웹사이트를 자동으로 조작합니다.
```
"GitHub에서 이슈 목록 가져와줘"
```

### call_subordinate (서브 에이전트)
하위 에이전트를 생성하여 작업을 위임합니다.
```
"프론트엔드와 백엔드를 동시에 작업해줘"
```

### memory_save / memory_load (메모리)
중요한 정보를 장기 기억에 저장하고 검색합니다.
```
"이 프로젝트 구조를 기억해줘"
```

### scheduler (스케줄러)
작업을 예약하거나 반복 실행합니다.
```
"매일 오전 9시에 서버 상태 체크해줘"
```

### response (응답)
사용자에게 최종 결과를 전달합니다.

### input (입력 요청)
추가 정보가 필요할 때 사용자에게 질문합니다.

## 멀티 에이전트 협업

Agent Zero의 핵심 기능 중 하나:

- **서브 에이전트 생성**: 복잡한 작업을 전문 에이전트에게 위임
- **에이전트 간 통신**: 정보 공유 및 작업 조율
- **계층 구조**: 상위 에이전트가 하위 에이전트에게 태스크 분배

---

## 프롬프트 엔지니어링 팁

1. **명확하고 구체적으로**: 원하는 결과를 정확히 기술
2. **컨텍스트 제공**: 배경 정보, 제약 조건, 원하는 형식 명시
3. **복잡한 작업은 분해**: 큰 작업을 작은 단위로 나누어 지시
4. **반복 개선**: 첫 시도에 완벽하지 않을 수 있음 — 결과 보고 지시 수정

## 음성 인터페이스

### TTS (Text-to-Speech)
- Preferences에서 Speech 토글 켜기
- 브라우저 내장 음성 합성기 사용 (로컬 처리)

### STT (Speech-to-Text)
- 마이크 버튼 클릭하여 음성 입력
- 로컬 Whisper 모델 사용 (외부 서버 전송 없음)
- Settings에서 모델 크기, 언어, 침묵 감지 조정 가능

## 백업 및 복원

Settings → Backup 탭에서:

### 백업 대상
- 지식베이스, 메모리, 채팅 히스토리
- 설정 파일, API 키
- 커스텀 도구, 업로드 파일

### 복원
- ZIP 파일 업로드
- 파일별 덮어쓰기/건너뛰기 선택 가능
- 다른 OS 간 호환

### 권장 백업 시점
- 대규모 변경 전
- 시스템 업데이트 전
- 프로젝트 마일스톤 달성 시
