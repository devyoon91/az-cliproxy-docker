## 기본 조작

### UI 액션 버튼

채팅 입력창 아래에 위치한 액션 버튼들:

| 버튼 | 기능 |
|------|------|
| **Pause/Resume Agent** | 에이전트 일시정지/재개 |
| **Import Knowledge** | 지식베이스에 파일 추가 (.txt, .pdf, .csv, .html, .json, .md) |
| **File Browser** | 파일 업로드/다운로드/삭제/탐색 |
| **Context** | LLM에 전송되는 전체 컨텍스트 확인 (디버깅용) |
| **History** | 대화 히스토리 JSON 확인 |
| **Nudge** | 에이전트가 멈췄을 때 마지막 프로세스 재시작 |
| **Restart** | 프레임워크 전체 재시작 (채팅 히스토리 유지) |
| **Skills** | v1.8 채팅 입력 메뉴에서 스킬 브라우징/활성화 |

### 파일 첨부

- 채팅 입력창 옆 📎 아이콘 클릭
- 여러 파일 동시 첨부 가능
- 이미지는 썸네일 미리보기 제공
- "이 파일들을 /a0/work_dir/project에 옮겨줘" 같은 지시 가능

### 파일 브라우저

- 디렉토리 탐색 (Up 버튼으로 상위 이동)
- 파일/폴더 업로드, 다운로드, 삭제
- 폴더는 zip으로 다운로드
- `/a0/work_dir`이 기본 작업 디렉토리

## 도구 사용법

### 기본 도구

| 도구 | 설명 | 예시 |
|------|------|------|
| `code_execution_tool` | Python/Node.js/Bash 코드 실행 | "Python으로 차트 만들어줘" |
| `search_engine` | SearXNG 기반 웹 검색 | "React 19 변경사항 검색해줘" |
| `document_query` | PDF, HTML, 웹 문서 분석 | "이 PDF에서 매출 추출해줘" |
| `browser_agent` | Playwright 브라우저 자동화 (vision 지원) | "GitHub 이슈 목록 가져와줘" |
| `call_subordinate` | 서브 에이전트 생성/위임 | "프론트엔드와 백엔드 동시에" |
| `memory_save` / `memory_load` | FAISS 기반 장기 기억 (SHA-256 검증) | "프로젝트 구조를 기억해줘" |
| `scheduler` | 작업 예약/반복 실행 (cron) | "매일 9시에 상태 체크해줘" |
| `text_editor` | 파일 편집 + 린팅 | "이 파일 3번째 줄 수정해줘" |
| `response` | 사용자에게 최종 응답 | - |
| `input` | 사용자에게 질문 | - |

### v1.8 신규 도구

| 도구 | 설명 | 예시 |
|------|------|------|
| `skills_tool:list` | 사용 가능한 스킬 목록 조회 | "어떤 스킬이 있어?" |

| `a2a_chat` | 원격 FastA2A 에이전트와 대화 | "외부 에이전트에게 물어봐줘" |
| `wait` | 지정 시간만큼 대기 | "5분 후 재시도해줘" |

## Skills 시스템 (v1.8 신규)

SKILL.md 표준 기반의 재사용 가능한 에이전트 스킬입니다. Claude Code, Cursor 등 다른 도구와 호환됩니다.

### 사용법

1. 채팅 입력 메뉴에서 스킬 브라우징
2. 또는 에이전트에게 지시: "사용 가능한 스킬 목록 보여줘"
3. 스킬 로드 후 해당 스킬의 지시에 따라 작업 수행

### 장점
- 스킬이 처음부터 활성화되어 토큰 절약
- 전문화된 워크플로우 즉시 사용

## Projects 시스템 (v1.8 신규)

프로젝트 컨텍스트를 관리하여 에이전트가 현재 작업 중인 프로젝트를 인식합니다.

- 활성/비활성 프로젝트 상태 관리
- 프로젝트별 설정 분리
- Git 기반 프로젝트 클론 및 관리

---

## Secrets 관리 (v1.8 신규)

민감한 정보를 안전하게 관리합니다.

- `§§secret(name)` 별칭으로 비밀값 참조
- 실제 값은 자동 주입, 프롬프트/로그에 노출되지 않음
- 환경변수, API 키 등을 안전하게 사용

---

## 원격 에이전트 통신 — A2A (v1.8 신규)

FastA2A 호환 원격 에이전트와 통신할 수 있습니다.

- `a2a_chat` 도구로 원격 에이전트에 메시지 전송
- 세션 컨텍스트 자동 유지
- 여러 에이전트 오케스트레이션 가능

---

## 멀티 에이전트 협업

- **서브 에이전트 생성**: 복잡한 작업을 전문 에이전트에게 위임
- **에이전트 간 통신**: 정보 공유 및 작업 조율
- **계층 구조**: 상위 에이전트가 하위 에이전트에게 태스크 분배
- **A2A**: 원격 에이전트와도 협업 가능 (v1.8)

---

## 프롬프트 엔지니어링 팁

1. **명확하고 구체적으로**: 원하는 결과를 정확히 기술
2. **컨텍스트 제공**: 배경 정보, 제약 조건, 원하는 형식 명시
3. **복잡한 작업은 분해**: 큰 작업을 작은 단위로 나누어 지시
4. **반복 개선**: 첫 시도에 완벽하지 않을 수 있음 — 결과 보고 지시 수정
5. **스킬 활용**: 관련 스킬이 있으면 직접 구현보다 효율적 (v1.8)

## 음성 인터페이스

### TTS (Text-to-Speech)
- Preferences에서 Speech 토글 켜기
- Kokoro TTS 모델 사용 (로컬)

### STT (Speech-to-Text)
- 마이크 버튼 클릭하여 음성 입력
- 로컬 Whisper 모델 사용 (외부 서버 전송 없음)
- Settings에서 모델 크기, 언어, 침묵 감지 조정 가능

## 보안

### _infection_check 플러그인
프롬프트 인젝션을 감지하는 내장 플러그인입니다.

- **기본: 꺼짐** (매 검사마다 추가 LLM 호출 → 토큰 비용 증가)
- 활성화: Plugins → `_infection_check` → Enable
- 외부 콘텐츠에서 악의적 지시를 감지하여 차단

### Secrets 시스템
- 민감한 정보는 `§§secret(name)` 별칭으로 안전하게 관리
- 프롬프트/로그에 실제 값 노출 방지
