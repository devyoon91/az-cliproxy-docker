# 비용 최적화 가이드

Agent Zero의 LLM API 토큰 사용량을 최소화하고 비용을 관리하는 전략입니다.

---

## 1. 모델 선택 전략

### 역할별 모델 배치

모든 작업에 최상위 모델을 쓸 필요 없습니다. 역할에 맞는 모델을 배치하면 비용을 크게 줄일 수 있습니다.

| 역할 | 추천 모델 | 비용 ($/1M 입력/출력) | 이유 |
|------|-----------|---------------------|------|
| **메인 모델** (chat) | o3, gpt-4.1 | $2.0 / $8.0 | 코딩, 설계, 복잡한 추론 |
| **유틸리티** (util) | gpt-4.1-mini | $0.4 / $1.6 | 요약, 압축, 메모리 검색 |
| **브라우저** (browser) | gpt-4.1-mini | $0.4 / $1.6 | 웹 탐색은 경량 모델로 충분 |
| **임베딩** (embed) | HuggingFace (로컬) | 무료 | 컨테이너 내장, API 호출 없음 |

**핵심**: 유틸리티와 브라우저 모델을 mini로 바꾸면 보조 작업 비용이 **80% 절감**됩니다.

### 모델별 비용 비교

| 모델 | 입력 ($/1M) | 출력 ($/1M) | 용도 |
|------|------------|------------|------|
| o3 | $2.00 | $8.00 | 최상위 추론/코딩 |
| gpt-4.1 | $2.00 | $8.00 | 범용 (o3과 동일 가격) |
| gpt-4.1-mini | $0.40 | $1.60 | 보조 작업 (유틸/브라우저) |
| gpt-4.1-nano | $0.10 | $0.40 | 초경량 (분류, 추출) |
| gpt-4o | $2.50 | $10.00 | 이전 세대 (비추천) |

## 2. Agent Zero 설정 최적화

### 컨텍스트 크기 줄이기

| 설정 | 기본값 | 권장값 | 효과 |
|------|--------|--------|------|
| `chat_model_ctx_length` | 100000 | **50000** | 매 호출마다 전송되는 토큰 절반 |
| `chat_model_ctx_history` | 0.7 | **0.5** | 대화 히스토리 비중 줄임 |

컨텍스트가 크면 매번 더 많은 토큰이 입력으로 들어갑니다. 50000이면 대부분의 작업에 충분합니다.

### 메모리 검색 빈도 줄이기

| 설정 | 기본값 | 권장값 | 효과 |
|------|--------|--------|------|
| `memory_recall_interval` | 3 | **5** | 메모리 검색 주기 늘림 (매 5턴) |
| `memory_recall_memories_max_search` | 12 | **6** | 검색 범위 절반 |

메모리 검색마다 유틸리티 모델이 호출됩니다. 간격을 늘리면 호출 횟수가 줄어듭니다.

### 플러그인 설정

| 플러그인 | 권장 | 이유 |
|----------|------|------|
| `_chat_compaction` | **켜짐** | 긴 대화를 요약하여 컨텍스트 절약 |
| `_infection_check` | **꺼짐** | 매 응답마다 추가 LLM 호출 발생 |
| `_error_retry` | **켜짐** | 실패 시 재시도 (낭비 방지) |

## 3. 프롬프트 작성 패턴

### 토큰을 아끼는 지시 방법

**나쁜 예** (모호함 → 여러 번 반복):
```
"뭔가 좋은 API 만들어줘"
→ 에이전트가 요구사항 질문 → 답변 → 또 질문 → 답변... (토큰 낭비)
```

**좋은 예** (구체적 → 한 번에 완료):
```
"FastAPI로 /api/users CRUD 만들어줘. 
SQLite 사용, Pydantic 모델은 id/name/email.
테스트 포함, /a0/work_dir/user-api에 저장."
```

### 서브 에이전트 활용

스케줄러 태스크 생성 시 `dedicated_context: true`를 설정하면, 태스크가 **별도 채팅**에서 실행되어 메인 대화의 컨텍스트가 커지지 않습니다.

```
dedicated_context: false (기본)          dedicated_context: true
─────────────────────────              ─────────────────────────
메인 채팅                               메인 채팅          태스크 채팅
├── 사용자 메시지                        ├── 사용자 메시지    ├── 태스크 작업
├── 에이전트 응답                        ├── "태스크 위임"    ├── 작업 수행...
├── 태스크 전체 대화 ← 여기 다 쌓임      ├── "결과: 요약"    └── 결과 반환
├── 다음 응답 ← 토큰 폭증               └── 컨텍스트 가벼움
└── ...
```

`call_subordinate`로 호출하는 서브 에이전트는 Agent Zero가 자동으로 컨텍스트를 관리합니다.

### 스킬 활용 (v1.8)

스킬을 로드하면 처음부터 전문화된 지시가 적용되어 불필요한 탐색/시행착오가 줄어듭니다.

```
"playwright 스킬 로드해서 브라우저 자동화해줘"
→ 스킬이 최적 워크플로우를 즉시 제공 (토큰 절약)
```

## 6. CLIProxy vs API 직접 사용 비용 비교

| 방식 | 월 비용 | 장점 | 단점 |
|------|---------|------|------|
| CLIProxy + 구독 | $20 (Plus) / $200 (Pro) | 정액제, 예측 가능 | 메시지 제한, 약관 리스크 (프로바이더별 상이) |
| API 직접 사용 | 사용량에 따라 변동 | 제한 없음, 합법 | 예측 어려움, 폭탄 가능 |
| 하이브리드 | 구독 + API | 기본은 구독, 초과 시 API | 설정 복잡 |

**권장**: API 직접 사용 + Hard Limit 설정이 가장 안전하고 유연합니다.
